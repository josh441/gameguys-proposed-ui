Golden rules the AI must respect
1
Surface, don’t rebuild. Xero, Nayax/Shopify sync, PDF PO reader, release calendar, supplier + task features already exist — reorganise and expose them.
2
Split source of truth. Nayax = sales & price truth (we pull/mirror). App = stock truth (deduct on fill), reconciled by the existing route reconciliation.
3
Deduct exactly once. Stock is withdrawn only at “Mark route complete” via the existing picklist_final_rows trigger. The filler hub must not add a second deduction (double-count risk).
4
Fillers are scoped. A filler only ever sees their assigned machines (individual assignments; routes are a bulk convenience).
5
Five tabs, max. Ownership · Inventory · Machines · Calendar · CRM. Everything deeper is a sub-tab. No stored supplier credentials.

Core flows

Filler fill workflow (flexible pick list)

Warehouse-aware quantities, out-of-stock handling, and the single authoritative deduction at the end.
flowchart TD
  A[Filler opens Machines tab] --> B[Sees ONLY assigned machines]
  B --> C[Tap Start Fill]
  C --> D[Suggested pick list from Nayax low-stock + par levels]
  D --> E{Enough warehouse stock per line?}
  E -- Yes --> F[Keep suggested qty]
  E -- Partial --> G[Auto-cap to available: wanted X, capped to Y]
  E -- Out of stock --> H[Skip / Substitute / Notify office]
  F --> I[Edit quantities with +/- stepper]
  G --> I
  H --> I
  I --> J[Add ad-hoc product if needed]
  J --> K[Print / PDF for poor reception]
  K --> L[Fill machine + set prices on Nayax]
  L --> M[Mark route complete]
  M --> N[Insert picklist_final_rows]
  N --> P[Trigger deducts warehouse stock ONCE]
  P --> Q[Reconciliation vs Nayax sales]
